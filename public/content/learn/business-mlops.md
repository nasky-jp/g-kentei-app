# MLOpsとAI開発プロセス

## 概要

**MLOps（Machine Learning Operations）** は、機械学習モデルの開発から本番運用までのライフサイクルを効率化・自動化する実践とツールの体系。DevOpsの原則をMLに適用したもので、「実験成功 ≠ 本番成功」という機械学習固有の課題に対処する。

---

## MLOpsの定義と構成要素

### MLライフサイクル

```
データ収集 → 特徴量エンジニアリング → モデル訓練 → 評価
      ↑                                              ↓
   モニタリング ← サービング ← 検証 ← パッケージング
```

### MLとDevOpsの違い

| 課題 | ソフトウェア開発 | 機械学習 |
|---|---|---|
| テスト | 単体・結合テスト | データ・モデル・パイプラインのテスト |
| バージョン管理 | コードのみ | コード + データ + モデル + 設定 |
| 再現性 | 環境を固定 | データ・コード・モデル全て一致が必要 |
| デプロイ | ビルド→リリース | 訓練→サービング（推論エンドポイント） |
| 品質劣化 | バグが原因 | データドリフトが原因 |

---

## CI/CD for ML

### CI（継続的インテグレーション）

- コードのコミット毎に自動テスト実行
- データバリデーション（スキーマ・統計チェック）
- モデルユニットテスト（推論速度・出力形式）

### CD（継続的デリバリー）

- 学習パイプラインの自動トリガー
- 評価指標をパスしたモデルのみデプロイ
- **ブルーグリーンデプロイ**・**カナリアリリース** で安全に切り替え

### CT（継続的トレーニング）

MLOps特有。以下のトリガーで自動再訓練：

- スケジュール（週次・月次）
- データドリフト検知
- 性能劣化アラート

---

## フィーチャーストア（Feature Store）

特徴量を中央集権的に管理・再利用するプラットフォーム：

### 役割

- 特徴量の計算ロジックをコード化して共有
- **トレーニング-サービングスキュー**を解消（同じ計算ロジックを使用）
- 特徴量の発見性向上（カタログ機能）

| 機能 | 説明 |
|---|---|
| オフラインストア | 訓練用の大量の履歴特徴量（Parquet・S3等） |
| オンラインストア | 推論用のリアルタイム特徴量（Redis・DynamoDB等） |
| バックフィル | 過去データの一括特徴量計算 |

代表的なFSS: Feast（OSSのデファクト）、Tecton、Vertex AI Feature Store

---

## モデルレジストリ

学習済みモデルをバージョン管理するリポジトリ：

| 管理情報 | 内容 |
|---|---|
| モデルファイル | weights・アーキテクチャ定義 |
| メタデータ | 訓練日時・訓練データのハッシュ・ハイパーパラメータ |
| 評価指標 | 精度・AUC・レイテンシ等 |
| ステージ | Staging → Production → Archived |

---

## モデルドリフト検知

本番環境でのモデルの性能劣化を監視する：

| ドリフト種別 | 内容 | 検知方法 |
|---|---|---|
| データドリフト | 入力データの分布変化 | KS検定・PSI（Population Stability Index） |
| コンセプトドリフト | 入出力の関係性が変化 | 予測精度の低下監視 |
| ラベルドリフト | 目的変数の分布変化 | ラベルの統計値変化 |

> ⚠️ **試験ポイント**
> データドリフトは「入力の分布変化」、コンセプトドリフトは「入出力関係の変化」。区別して理解する。

---

## A/Bテスト

複数のモデルを実際のユーザーに提供して効果を比較：

```
ユーザー
  ├─ 50%：モデルA（コントロール）
  └─ 50%：モデルB（チャレンジャー）
```

- 統計的有意差を確認してから全面切り替え
- **シャドウモード**：Bは実際には影響を与えず評価のみ

## 代表的なMLOpsツール

| ツール | カテゴリ | 特徴 |
|---|---|---|
| MLflow | 実験管理・レジストリ | OSS・軽量・エコシステムが広い |
| Kubeflow | パイプライン・Kubernetes上のML | Google発・スケーラブル |
| Airflow | パイプラインオーケストレーション | 汎用DAGスケジューラ |
| DVC | データバージョン管理 | Gitライクなデータ管理 |
| Feast | フィーチャーストア | OSS・軽量・シンプル |
| Seldon/BentoML | モデルサービング | REST/gRPCエンドポイント提供 |

## 関連用語

| 用語 | 意味 |
|---|---|
| MLOps | ML開発→本番運用のライフサイクル効率化 |
| フィーチャーストア | 特徴量を中央管理・再利用するプラットフォーム |
| データドリフト | 本番データの統計的分布が訓練時から変化する現象 |
| コンセプトドリフト | 入力と出力の関係性が時間とともに変化する現象 |
| モデルレジストリ | 学習済みモデルをバージョン管理するリポジトリ |
| トレーニング-サービングスキュー | 訓練時と推論時の特徴量計算の不一致 |
