# モデル評価

## 概要

機械学習モデルの性能を正しく測ることは、実用化において極めて重要です。評価指標の選択を誤ると、「数字は良いが現場で使えないモデル」になります。

---

## 分類問題の評価指標

### 混同行列（Confusion Matrix）

|  | 予測: 正 | 予測: 負 |
|---|---|---|
| **実際: 正** | TP（真陽性） | FN（偽陰性） |
| **実際: 負** | FP（偽陽性） | TN（真陰性） |

### 主要指標

| 指標 | 計算式 | 意味 |
|---|---|---|
| **精度（Accuracy）** | (TP+TN) / 全体 | 全体の正解率 |
| **適合率（Precision）** | TP / (TP+FP) | 「正」と予測したうち本当に正の割合 |
| **再現率（Recall）** | TP / (TP+FN) | 実際の「正」をどれだけ検出できたか |
| **F1スコア** | 2×P×R / (P+R) | 適合率と再現率の調和平均 |

### 指標の使い分け

| シーン | 重視する指標 | 理由 |
|---|---|---|
| スパムフィルター | **適合率** | 正常メールをスパムと誤判定したくない |
| がん検診 | **再現率** | 見逃し（FN）を最小化したい |
| クラス不均衡 | **F1スコア** | Accuracyは多数派クラスに偏る |

---

## 回帰問題の評価指標

| 指標 | 説明 | 特徴 |
|---|---|---|
| **MAE**（平均絶対誤差） | 誤差の絶対値の平均 | 外れ値に頑健 |
| **MSE**（平均二乗誤差） | 誤差の二乗の平均 | 外れ値の影響大 |
| **RMSE** | MSEの平方根 | 元の単位で解釈可能 |
| **R²（決定係数）** | 分散の説明割合（0〜1） | 1に近いほど良い |

---

## 過学習（Overfitting）

**訓練データに特化しすぎて、未知データに対する性能が低い状態**。

```
過学習: 訓練精度↑ / テスト精度↓
```

### 対策

| 手法 | 説明 |
|---|---|
| **正則化** | パラメータが大きくなりすぎないようペナルティを加える |
| **ドロップアウト** | ランダムにニューロンを無効化（DL向け）|
| **早期停止** | 検証誤差が増え始めたら学習を止める |
| **データ拡張** | 学習データを人工的に増やす |

---

## 交差検証（Cross-Validation）

データを分割して複数回評価し、より信頼性の高い性能推定を行う手法。

### k分割交差検証

1. データをk個に分割
2. 1つをテスト用、残りを訓練用として評価（k回繰り返す）
3. k回の評価の平均を最終スコアとする

一般的には **k=5** または **k=10** が使われる。

---

## 試験ポイント

> ⚠️ G検定では以下が頻出です

- **適合率 vs 再現率**のトレードオフと使い分け
- **F1スコアは調和平均**（単純平均ではない）
- **過学習の原因と対策**（正則化・ドロップアウト等）
- **交差検証の目的**：汎化性能の推定
- クラス不均衡データでは **Accuracy が使えない** ことがある

## 関連用語

| 用語 | 説明 |
|---|---|
| 汎化性能 | 未知データへの予測性能 |
| 過学習 | 訓練データに過度に適合した状態 |
| 未学習 | モデルが単純すぎて訓練データも学習できない状態 |
| ROC曲線 | TPRとFPRのトレードオフを可視化 |
| AUC | ROC曲線の面積。1に近いほど良い |
