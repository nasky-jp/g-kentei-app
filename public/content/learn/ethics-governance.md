# AIガバナンスと原則

## 概要

AIの急速な普及に伴い、倫理的・社会的リスクを管理するための**AIガバナンス**フレームワークが国内外で整備されつつある。G検定ではOECD原則・EU AI Act・内閣府のAI戦略など主要な指針の内容と動向が問われる。

---

## 主要なAI原則

### OECD AI原則（2019年）

OECDが策定した最初の国際的なAI原則。G7/G20でも採択：

1. **包摂的成長・持続可能な発展・幸福**：AIは人々の幸福に貢献すべき
2. **人間中心の価値観と公平性**：人権・民主的価値の尊重
3. **透明性と説明可能性**：AIシステムの意思決定を説明できること
4. **堅牢性・セキュリティ・安全性**：リスク管理と継続的な安全確保
5. **説明責任**：AIシステムの結果に対する責任の明確化

### 内閣府「人間中心のAI社会原則」（2019年）

| 原則 | 内容 |
|---|---|
| 人間の尊厳 | AIは人間の自律性を損なってはならない |
| 多様性・包摂性 | 社会の多様性に配慮した設計 |
| 持続可能性 | 地球環境・社会の持続可能性 |
| 安全 | 身体的・精神的安全の確保 |
| セキュリティ | 悪意ある使用への対策 |
| プライバシー | 個人情報・プライバシーの保護 |
| 公平性 | 差別・偏見のない公正な判断 |
| 透明性 | 説明可能・検証可能なシステム |
| イノベーション | 研究開発の推進と国際協調 |

> ⚠️ **試験ポイント**
> OECD原則の5項目と内閣府の9原則の大枠を理解する。「透明性」「説明責任」「公平性」は共通して重要な概念。

## EU AI Act（EU AI規制法）

2024年施行。世界初の包括的なAI規制：

### リスクベースアプローチ

AIシステムをリスクレベルで4段階に分類：

| リスクレベル | 内容 | 規制 |
|---|---|---|
| 容認不可能 | 人の意識を操作するAI・リアルタイム顔認識（公共） | **禁止** |
| 高リスク | 医療診断・採用・信用スコアリング・自動運転 | 適合性評価・登録義務 |
| 限定リスク | チャットボット・ディープフェイク | 透明性義務（AIと明示） |
| 最小リスク | スパムフィルタ・AIゲーム | 自主規制のみ |

> ⚠️ **試験ポイント**
> EU AI Actはリスクベースアプローチを採用。「容認不可能リスク」が禁止されている点と、高リスクAIの義務を理解する。

## AIの透明性・説明責任・公正性

### 説明可能AI（XAI：Explainable AI）

ブラックボックスなAIの判断根拠を人間が理解できるようにする技術：

| 手法 | 概要 |
|---|---|
| LIME | 局所的な線形近似でAIの判断を説明 |
| SHAP | Shapley値を用いた特徴量寄与度の説明 |
| Attention可視化 | Transformerの注意機構の可視化 |
| Grad-CAM | CNNが画像のどの部分を見ているかを可視化 |

### アルゴリズム公正性の指標

| 指標 | 定義 |
|---|---|
| 人口統計学的等価性 | 各グループでの陽性予測率が等しい |
| 機会均等 | 各グループでの真陽性率（再現率）が等しい |
| 較正（Calibration） | 予測確率が実際の頻度と一致している |

## AIインシデント管理

### AIインシデントの事例

- 採用AIの性別差別（Amazon、2018年）
- 顔認識AIの人種間精度格差（NIST調査、2019年）
- 自動運転車の事故（Uber、Waymo等）
- GPT系モデルの有害コンテンツ生成

### インシデント対応の枠組み

1. **検出**：モニタリング・アラート設定
2. **評価**：影響範囲・深刻度のアセスメント
3. **対応**：システム停止・パッチ適用・通知
4. **報告**：規制当局・利用者への開示
5. **学習**：事後レビュー・再発防止策

## グローバルな規制動向

| 地域 | 規制・指針 | 特徴 |
|---|---|---|
| EU | EU AI Act（2024） | 包括的規制。リスク分類ベース |
| 米国 | AI権利章典（2022）・大統領令（2023） | 任意原則ベース・省庁横断 |
| 中国 | 生成AI規制（2023）・アルゴリズム推薦規制 | 安全評価・実名登録義務 |
| 日本 | AI事業者ガイドライン（2024）・AI戦略会議 | 自主規制・国際協調重視 |
| 国際 | G7広島AIプロセス（2023）・OECD AI原則 | 国際標準・相互運用性 |

## リスク管理フレームワーク

### NIST AI RMF（米国標準技術研究所）

AIリスク管理のための4つの機能：

1. **GOVERN**：組織全体のAIリスクガバナンスの確立
2. **MAP**：AIリスクの特定とコンテキスト化
3. **MEASURE**：AIリスクの分析と評価
4. **MANAGE**：AIリスクの優先付けと対応
