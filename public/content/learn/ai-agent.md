# AIエージェントと強化学習概論

## 概要

**AIエージェント**とは、環境を知覚し、目標達成に向けて自律的に行動するソフトウェアである。強化学習はエージェントが試行錯誤を通じて最適な行動方策を学習するフレームワークで、ゲームAI・ロボット制御・推薦システムなどで活用される。

---

## エージェントの基本構造

### 知覚→行動ループ

```
環境（Environment）
  ↓ 状態（State）・報酬（Reward）
エージェント（Agent）
  ↓ 行動（Action）
環境（Environment）
```

### エージェントの種類

| 種類 | 特徴 | 例 |
|---|---|---|
| 単純反射エージェント | 現在の知覚だけで行動 | ルームバ（単純版） |
| モデルベースエージェント | 内部状態モデルを保持 | 自動運転 |
| 目標ベースエージェント | 目標状態に向けて計画 | チェスAI |
| 効用ベースエージェント | 期待効用を最大化 | 投資アルゴリズム |
| 学習エージェント | 経験から改善 | AlphaGo |

## マルコフ決定過程（MDP）

強化学習の数学的基盤。以下の要素で定義される：

| 要素 | 記号 | 説明 |
|---|---|---|
| 状態集合 | S | エージェントが取り得る状態の集合 |
| 行動集合 | A | エージェントが取り得る行動の集合 |
| 遷移確率 | P(s'|s,a) | 状態sで行動aをとったときs'に遷移する確率 |
| 報酬関数 | R(s,a) | 状態sで行動aをとったときの即時報酬 |
| 割引率 | γ（0≦γ≦1） | 将来報酬の現在価値への割引 |

### マルコフ性

> **次の状態は現在の状態のみに依存し、過去の履歴には依存しない**

これをマルコフ性（Markov Property）と呼ぶ。

---

## 報酬・価値関数・方策

### 累積報酬（Return）

$$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$$

### 価値関数

- **状態価値関数** V(s)：状態sから方策πに従ったときの期待累積報酬
- **行動価値関数** Q(s,a)：状態sで行動aをとり、その後方策πに従ったときの期待累積報酬

### 方策（Policy）

- **決定論的方策**：π(s) = a（状態sで必ず行動aをとる）
- **確率論的方策**：π(a|s)（状態sで行動aをとる確率）
- **最適方策** π\*：すべての状態でV(s)を最大化する方策

> ⚠️ **試験ポイント**
> MDPの要素（S・A・P・R・γ）を覚える。γ=0は即時報酬のみ評価、γ=1は無限先まで等価評価。

---

## Q学習の直感

Q学習はモデルフリーの強化学習アルゴリズム。行動価値関数Q(s,a)をベルマン方程式で更新する。

**更新式：**
```
Q(s,a) ← Q(s,a) + α[R + γ·max_a' Q(s',a') - Q(s,a)]
```

- α：学習率
- γ：割引率
- `R + γ·max_a' Q(s',a')`：TD目標（temporal difference target）
- `R + γ·max_a' Q(s',a') - Q(s,a)`：TDエラー

---

## マルチエージェントシステム

- 複数のエージェントが同一環境で協調・競争
- **協調型**：分散センサー・自動倉庫（Kiva）
- **競争型**：ゲームAI・市場シミュレーション
- **混合型**：StarCraft II AI（OpenAI Five）

> ⚠️ **試験ポイント**
> 強化学習の基本要素：エージェント・環境・状態・行動・報酬・方策・価値関数。マルコフ性の定義は頻出。

## 関連用語

| 用語 | 意味 |
|---|---|
| エージェント | 環境を知覚し自律的に行動するシステム |

| 方策（Policy） | 状態から行動へのマッピング |

| 割引率γ | 将来報酬をどれだけ割り引くかのパラメータ |
| ベルマン方程式 | 価値関数の再帰的な関係式 |
