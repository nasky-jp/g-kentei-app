# 学習の最適化テクニック

## 概要

ニューラルネットワークの学習では、損失関数を最小化するためにパラメータを更新する。**最適化アルゴリズム・正則化・正規化**の選択が学習の速度・安定性・汎化性能を大きく左右する。

---

## 最適化アルゴリズム

### SGD（Stochastic Gradient Descent）

- ミニバッチごとに勾配を計算してパラメータ更新
- **シンプルだが収束が遅い・鞍点に引っかかりやすい**
- モーメンタム（SGD+M）で慣性を加えて改善

```
θ ← θ - η∇L(θ)
```

### Adam（Adaptive Moment Estimation）

- 1次モーメント（勾配の平均）と2次モーメント（勾配の分散）を適応的に使用
- 各パラメータごとに学習率を自動調整
- バイアス補正でステップ初期の不安定さを解消

```
m ← β₁m + (1-β₁)g     # 1次モーメント
v ← β₂v + (1-β₂)g²    # 2次モーメント
θ ← θ - η·m̂/（√v̂ + ε）
```

### RMSprop

- 勾配の2乗移動平均で学習率を調整
- AdamのAdaptiveな部分のみ（1次モーメントなし）
- RNNの学習に有効

| 比較 | SGD | Adam | RMSprop |
|---|---|---|---|
| 収束速度 | 遅い | 速い | 速い |
| 汎化性能 | 高い（最終的に） | 中 | 中 |
| ハイパーパラメータ調整 | 難しい | 容易 | 中 |

> ⚠️ **試験ポイント**
> Adamは多くのタスクでデフォルトとして有効。最終精度はSGD+Mが勝ることもある（画像分類等）。

---

## 学習率スケジューリング

固定学習率では最適解に到達しにくい場合に、エポックに応じて変化させる：

| スケジューラ | 動作 |
|---|---|
| Step Decay | 一定エポックごとに学習率を減衰 |
| Cosine Annealing | コサイン曲線で滑らかに減衰（再スタートあり） |
| Warmup | 最初は低学習率から徐々に増加（Transformerで頻用） |
| Cyclical LR | 周期的に学習率を増減させ局所最適を脱出 |
| OneCycle | Warmup→最大→急速減衰の1サイクル |

## 正規化（Normalization）

### バッチ正規化（Batch Normalization）

- ミニバッチ内で各特徴量を正規化（平均0・分散1）
- **学習を安定化・収束を加速・正則化効果もある**
- γ・βの学習可能パラメータでスケーリング・シフト
- **問題**：バッチサイズが小さいと不安定（RNNにも不向き）

### 層正規化（Layer Normalization）

- サンプル内の全特徴量を正規化（バッチ方向ではなく特徴量方向）
- **Transformerで標準**。バッチサイズに依存しない
- RNN・BERT・GPTなどで使用

| 正規化手法 | 正規化の軸 | 主な用途 |
|---|---|---|
| Batch Norm | バッチ方向（特徴量ごと） | CNN・MLP |
| Layer Norm | 特徴量方向（サンプルごと） | Transformer・RNN |
| Instance Norm | 特徴量・空間方向（サンプルごと） | スタイル変換 |
| Group Norm | チャネルをグループ化して正規化 | 小バッチのCNN |

> ⚠️ **試験ポイント**
> Batch Normはバッチ方向、Layer NormはTransformerで使われる。両者の違いと使用場面を対応させて覚える。

---

## 正則化（Regularization）

過学習を防ぐためにモデルの複雑さにペナルティを課す手法。

### L1正則化（Lasso）

- 損失関数に |w| の和を加算
- **スパース解**（多くの重みが0になる）→ 特徴選択に使える

### L2正則化（Ridge / Weight Decay）

- 損失関数に w² の和を加算
- 重みを0に近づける（0にはならない）
- Adam では Weight Decay（AdamW）として実装されることが多い

### ドロップアウト（Dropout）

- 学習時にランダムにニューロンを無効化（確率p）
- **アンサンブル効果**：毎回異なるサブネットを学習
- 推論時は全ニューロンを使い重みをスケール（または学習時にスケール補正）
- Transformerでは Attention Dropout として使用

| 正則化 | 効果 | 特性 |
|---|---|---|
| L1 | スパース化 | 特徴選択・係数が0になる |
| L2 | 重みの均等化 | 係数が小さくなる（0にはなりにくい） |
| Dropout | アンサンブル効果 | ランダムなサブネット学習 |

---

## 勾配クリッピング（Gradient Clipping）

- 勾配のノルムが閾値を超えたら制限する
- **勾配爆発（Gradient Exploding）** を防ぐ
- RNNや深いネットワークで特に有効

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

---

## 混合精度学習（Mixed Precision Training）

- FP32（32bit浮動小数）と FP16（16bit半精度）を混在させて学習
- **メモリ削減・計算高速化**（GPU の Tensor Core 活用）
- **Loss Scaling**で FP16 の数値的安定性を確保
- PyTorchでは `torch.cuda.amp` で簡単に有効化

## 関連用語

| 用語 | 意味 |
|---|---|
| 学習率（η） | パラメータ更新の大きさを制御するハイパーパラメータ |
| モーメンタム | 過去の勾配方向を慣性として利用して収束を安定化 |
| 勾配爆発 | 深いネットワークで勾配が指数的に大きくなる問題 |
| 勾配消失 | 深いネットワークで勾配が消えてしまう問題 |
| Weight Decay | L2正則化をAdamに組み込んだもの（AdamW） |
| FP16 | 16ビット半精度浮動小数。メモリ効率が良い |
