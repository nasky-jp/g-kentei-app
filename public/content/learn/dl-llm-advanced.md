# LLM応用・RAG・プロンプト技術

## 概要

大規模言語モデル（LLM）を実用システムに組み込むための応用技術を扱う。RAGによる外部知識統合、プロンプトエンジニアリング、効率的なファインチューニング手法（LoRA）、人間フィードバックによる強化学習（RLHF）、そしてLLMエージェントの構成を理解する。

---

## RAG（Retrieval-Augmented Generation）

外部知識データベースから関連文書を検索し、LLMのプロンプトに付与して回答を生成する手法。

**構成要素**

| コンポーネント | 役割 |
|---|---|
| Retriever（検索器） | クエリに関連する文書をベクトルDBなどから取得 |
| Generator（生成器） | 取得した文書をコンテキストとしてLLMが回答を生成 |
| ベクトルストア | 文書をEmbeddingで索引化して保管（Pinecone・FAISS等） |

**メリット**
- ハルシネーション（事実誤認）の低減
- 最新情報・ドメイン固有知識への対応
- モデル再学習なしで知識を更新可能

---

## プロンプトエンジニアリング

LLMへの入力（プロンプト）を設計して出力品質を高める技術。

| 手法 | 説明 | 用途 |
|---|---|---|
| Zero-shot | 例示なしでタスクを指示 | 汎用タスク |
| Few-shot | 入出力の例を複数提示 | 形式が決まったタスク |
| Chain-of-Thought（CoT） | 「ステップごとに考えてください」と推論過程を促す | 数学・論理推論 |
| Self-Consistency | CoTを複数回実行して多数決を取る | 精度向上 |

---

## LoRA（Low-Rank Adaptation）

大規模モデルの効率的ファインチューニング手法。元の重みを固定し、低ランクの行列ペア（A・B）を追加して学習する。

- **QLoRA**: LoRAを量子化（4bit）と組み合わせてVRAM消費を大幅削減
- **Adapter**: ネットワーク層間に小さなモジュールを挿入する類似手法
- **PEFT（Parameter-Efficient Fine-Tuning）**: LoRA・Adapter等の省パラメータ学習手法の総称

**フルファインチューニングとの比較**

| 項目 | フルFT | LoRA |
|---|---|---|
| 学習パラメータ数 | 全パラメータ | 追加した低ランク行列のみ（1〜10%程度） |
| 必要なVRAM | 大 | 小 |
| 元モデルの保持 | 上書き | 元モデルは固定のまま |
| 切り替え | 不可 | アダプタの差し替えで複数タスク対応可 |

---

## RLHF（Reinforcement Learning from Human Feedback）

人間のフィードバックを用いてLLMの出力を人間の好みに合わせる学習手法。ChatGPT・InstructGPTで採用。

**3ステップのプロセス**

1. **SFT（Supervised Fine-Tuning）**: 高品質な人間の回答でモデルをファインチューニング
2. **報酬モデルの学習**: 複数の回答を人間がランク付けし、好ましさを予測する報酬モデルを訓練
3. **PPO（Proximal Policy Optimization）**: 報酬モデルのスコアを最大化するようにLLMをRL学習

---

## LLMエージェント

LLMが外部ツールを呼び出したり、複数ステップにわたって計画・実行するシステム。

| 概念 | 説明 |
|---|---|
| ツール使用 | 検索・コード実行・APIコールをLLMが呼び出す |
| ReAct | Reasoning（推論）とAction（行動）を交互に繰り返すフレームワーク |
| プランニング | タスクをサブゴールに分解して順次実行 |
| マルチエージェント | 複数のエージェントが役割分担して協調 |

---

## 基盤モデルとスケーリング則

**基盤モデル（Foundation Model）**: 大規模データで事前学習し、様々なタスクに転用できる大規模モデル（GPT-4・Gemini・Claude等）。

**スケーリング則（Scaling Law）**: モデルサイズ・データ量・計算量を増やすと性能が予測可能なスケールで向上するという経験則。

---

> ⚠️ **試験ポイント**
> - RAGの2大構成要素は **Retriever（検索器）** と **Generator（生成器）**
> - RLHFの3ステップ: **SFT → 報酬モデル学習 → PPO**（順番を問われる）
> - LoRAはフルファインチューニングと異なり **元の重みを固定** したまま低ランク行列を追加学習する
> - Few-shotはタスク例を提示、Zero-shotは例示なし、CoTは推論ステップを明示させる

## 関連用語

| 用語 | 説明 |
|---|---|
| RAG | 外部検索と生成を組み合わせる手法 |
| ハルシネーション | LLMが事実ではない情報を生成してしまう現象 |
| LoRA | 低ランク行列を追加するPEFT手法 |
| QLoRA | LoRAと量子化を組み合わせた省メモリFT手法 |
| PEFT | 省パラメータファインチューニング手法の総称 |
| RLHF | 人間フィードバックによる強化学習 |
| SFT | 教師データによる教師ありファインチューニング |
| PPO | RLHFで使われる近接方策最適化アルゴリズム |
| ReAct | 推論と行動を交互に行うエージェントフレームワーク |
| スケーリング則 | モデル規模と性能の予測可能な比例関係 |
| 基盤モデル | 大規模事前学習を経た汎用大規模モデル |
| CoT | 推論ステップを明示させるプロンプト技術 |
