# 強化学習

## 概要

**強化学習（Reinforcement Learning, RL）** は、エージェントが環境との相互作用を通じて試行錯誤し、累積報酬を最大化する行動方策を学習するパラダイム。教師あり学習とは異なり、正解ラベルではなく報酬シグナルから学習する。

---

## 探索と活用のトレードオフ

強化学習における根本的なジレンマ：

| 戦略 | 説明 | リスク |
|---|---|---|
| **活用（Exploitation）** | 現在最も報酬が高い行動を選ぶ | 局所最適に陥る |
| **探索（Exploration）** | 未知の行動を試す | 短期的に報酬が下がる |

### バランス調整手法

- **ε-greedy法**：確率εでランダム行動、1-εで最良行動
- **UCB（Upper Confidence Bound）**：不確実性が高い行動を優先
- **ボルツマン探索**：温度パラメータTで探索度合いを制御

> ⚠️ **試験ポイント**
> ε-greedy法は最も基本的な探索戦略。εを徐々に減らすことで学習初期は探索、後期は活用に移行できる。

## Q学習・SARSA・TD学習

### 時間的差分学習（TD学習）

即時報酬と推定価値の差（TDエラー）を使って逐次更新する：

```
V(s) ← V(s) + α[R + γV(s') - V(s)]
```

### Q学習（Off-policy）

次状態での**最大**Q値を使って更新：

```
Q(s,a) ← Q(s,a) + α[R + γ·max_a' Q(s',a') - Q(s,a)]
```

- **Off-policy**：更新に使うデータと実際の方策が異なっていても良い
- 最適行動価値関数 Q\* に収束する（理論的保証あり）

### SARSA（On-policy）

次状態での**実際に選んだ行動**のQ値を使って更新：

```
Q(s,a) ← Q(s,a) + α[R + γQ(s',a') - Q(s,a)]
```

- **On-policy**：自分が実際に従っている方策で更新
- SARSAは State-Action-Reward-State-Action の略

| 比較 | Q学習 | SARSA |
|---|---|---|
| 方策 | Off-policy | On-policy |
| 安全性 | 低（崖では転落しやすい） | 高（崖を避けて歩く） |
| 最適性 | Q\*に収束 | 現在の方策の価値に収束 |

## DQN（Deep Q-Network）

DeepMindが2015年に発表。Atariゲームで人間を超えた性能を達成。

### Q学習の限界

- 状態空間が大きい場合、Qテーブルをメモリに格納できない

### DQNの工夫

1. **ニューラルネットワークでQ関数を近似**：状態sを入力、各行動のQ値を出力
2. **経験再生（Experience Replay）**：過去の経験をバッファに保存しランダムサンプリング→相関を除去
3. **ターゲットネットワーク**：更新対象と目標Q値の計算を分離→学習を安定化

> ⚠️ **試験ポイント**
> DQNの3つの工夫（ニューラルネット近似・経験再生・ターゲットネット）は頻出。

## 方策勾配法（Policy Gradient）

価値関数ではなく**方策そのもの**をパラメータ化して勾配上昇法で最適化：

$$\nabla_\theta J(\theta) = \mathbb{E}_\pi \left[ \nabla_\theta \log \pi_\theta(a|s) \cdot G_t \right]$$

- **REINFORCE**：最もシンプルな方策勾配アルゴリズム
- 連続行動空間に対応可能
- 分散が大きく学習が不安定になりやすい

### Actor-Critic

- **Actor**：方策πを学習（何をすべきか）
- **Critic**：価値関数Vを学習（どれだけ良い状態か）
- ACの発展形：A2C・A3C・PPO・SAC

| アルゴリズム | 特徴 |
|---|---|
| A3C | 非同期並列学習。複数Workerで探索 |
| PPO | Clipにより安定した更新。OpenAIが推奨 |
| SAC | 最大エントロピー強化学習。ロボット制御で強い |

## AlphaGo・AlphaZero

| 手法 | 概要 |
|---|---|
| AlphaGo（2016） | 人間の棋譜で教師あり学習→自己対局で強化学習。モンテカルロ木探索（MCTS）組み合わせ |
| AlphaGo Zero（2017） | 人間の棋譜なし。完全自己対局のみで世界最強到達 |
| AlphaZero（2018） | チェス・将棋・囲碁に汎化。ゲームルールのみ与える |
| MuZero（2020） | ゲームのルールも学習。モデルベースRL |

> ⚠️ **試験ポイント**
> AlphaGoは教師あり学習+強化学習+MCTSの組み合わせ。AlphaGo Zeroは人間の棋譜を使わずに超えた点が革新的。

## 関連用語

| 用語 | 意味 |
|---|---|
| TDエラー | 予測価値と更新目標の差分 |
| 経験再生 | 過去の経験をランダムサンプリングして学習相関を除去 |
| ターゲットネット | 学習安定化のためQ値計算を担う固定ネットワーク |
| MCTS | モンテカルロ木探索。ゲームAIで使われる探索手法 |
| Actor-Critic | 方策と価値関数を同時学習するフレームワーク |
| PPO | 安定した方策更新を行う人気のRLアルゴリズム |
